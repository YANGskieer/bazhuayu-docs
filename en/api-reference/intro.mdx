# Octoparse OpenAPI

> Octoparse OpenAPI 参考 — 涵盖认证、任务管理、云端提取及数据导出。

使用 Octoparse OpenAPI 以编程方式管理任务、运行云端提取并获取采集数据。

## 开始之前

需要一个**标准版、专业版或企业版**账户，且账户中至少有一个可运行的任务。

## 基础 URL

```
https://openapi.octoparse.com
```

## 速率限制

**每秒 20 个请求**。如果收到 `429` 状态码，请降低请求频率。

## 认证

- [获取新 Token](/en/api-reference/access-token/obtain-a-new-token) — 所有用户
- [刷新 Token](/en/api-reference/access-token/refresh-token) — 所有用户

## API 端点

### 云端提取（专业版及以上）
| 端点 | 说明 |
|---|---|
| 启动任务 | 触发云端提取 |
| 停止任务 | 取消正在运行的提取 |
| 获取任务状态 | 查看提取进度 |
| 获取任务状态 V2 | 改进版进度追踪 |
| 启动/停止子任务 | 管理批量子任务 |

### 任务管理（标准版及以上）
| 端点 | 说明 |
|---|---|
| 搜索任务 | 按条件查找任务 |
| 复制任务 | 克隆已有任务 |
| 移动任务 | 迁移任务位置 |
| 更新任务参数 | 修改任务设置 |

### 数据导出（标准版及以上）
| 端点 | 说明 |
|---|---|
| 获取未导出数据 | 检索未导出的记录 |
| 按偏移量获取数据 | 分页数据检索 |
| 标记数据为已导出 | 将记录标记为已导出 |
| 删除数据 | 删除已采集记录 |

## 故障排查

每次响应均包含 `requestId`。如请求失败，请将其提供给 [Octoparse 支持团队](mailto:support@octoparse.com)。
