# 获取 API Key

> 适用于 Coze / Dify 自定义 MCP、企业微信智能机器人、QClaw、Cursor、Claude Code 等不支持 OAuth 或需固定 Key 的场景。

## 何时使用 API Key

| 认证方式 | 适用场景 |
| --- | --- |
| **OAuth 2.1** | ChatGPT、Claude、Cursor、Gemini 等支持浏览器登录的客户端；Coze、Dify 安装官方工具时 |
| **API Key** | Coze 自定义 MCP、Dify API Key 模式、企业微信、QClaw、Cursor、Claude Code、脚本 / 无头环境 |

使用 API Key 时，请求头需携带：

```text
x-api-key: 你的APIKey
```

服务地址（八爪鱼）：

```text
https://mcp.bazhuayu.com
```

国际 Octoparse 环境可使用 `https://mcp.octoparse.com`。

## 创建 API Key

1. 登录八爪鱼账户（[八爪鱼官网](https://www.bazhuayu.com/) 或 [Octoparse 账户中心](https://www.octoparse.com/console/account-center/api-keys)）。
2. 进入 **账户中心 → API Keys**（或「开发者 / API 密钥」）。
3. 点击 **创建 API Key**，按提示命名（如 `MCP-Cursor`、`MCP-Coze`）。
4. **立即复制并妥善保存** Key；部分界面仅展示一次，关闭后无法再次查看完整密钥。
5. 若 Key 泄露或员工离职，请作废旧 Key 并新建。

<Warning>
  不要将 API Key 提交到 Git、飞书文档公开页、截图或客服群。仅在 MCP 客户端的「密钥 / Headers」配置项中填写。
</Warning>

## 在 MCP 客户端中配置

### 通用 HTTP MCP 配置

```json
{
  "type": "http",
  "url": "https://mcp.bazhuayu.com",
  "headers": {
    "x-api-key": "你的APIKey"
  }
}
```

### Cursor 示例（`~/.cursor/mcp.json`）

```json
{
  "mcpServers": {
    "octoparse": {
      "type": "http",
      "url": "https://mcp.bazhuayu.com",
      "headers": {
        "x-api-key": "你的APIKey"
      }
    }
  }
}
```

### Claude Code 示例（`~/.claude/settings.json`）

```json
{
  "mcpServers": {
    "octoparse": {
      "type": "http",
      "url": "https://mcp.bazhuayu.com",
      "headers": {
        "x-api-key": "你的APIKey"
      }
    }
  }
}
```

保存后重启客户端，在 MCP 工具列表中应能看到八爪鱼相关工具。

更多客户端（ChatGPT、Gemini、OpenClaw、VS Code）见 [客户端配置指南](/en/mcp/guides/clients)。

## 验证是否生效

连接成功后，可在对话中尝试：

- 「用八爪鱼搜索一个采集新闻标题的模板」
- 「列出我账户下的八爪鱼任务」

若返回 `401` / `403`，请检查 Key 是否有效、是否填在 `x-api-key` 字段，以及 URL 是否正确。
