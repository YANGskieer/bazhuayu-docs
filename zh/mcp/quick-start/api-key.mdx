## 何时使用 API Key

| 认证方式 | 适用场景 |
| --- | --- |
| **OAuth 2.1** | ChatGPT、Claude、Cursor、Gemini 等支持浏览器登录的客户端；Coze、Dify 安装官方工具时 |
| **API Key** | Coze 自定义 MCP、Dify API Key 模式、QClaw、Cursor、Claude Code、脚本 / 无头环境 |

<Warning>
  不要将 API Key 提交到 Git、飞书文档公开页、截图或客服群。仅在 MCP 客户端的「密钥 / Headers」配置项中填写。
</Warning>

## 方式一：交给 Agent 自动配置（推荐）

WorkBuddy、QClaw、豆包专业版等能够管理工具配置的 Agent，可以自行选择当前客户端支持的配置方式、写入 MCP 设置并测试连接。将下面的内容发给 Agent，并把占位符替换为刚刚创建的 Key：

```text
请帮我添加名为“八爪鱼”的 HTTP MCP 服务，并自行完成配置和连通性测试。

服务地址：https://mcp.bazhuayu.com
API Key：<粘贴你的 API Key>

关键要求：
1. 本次接入只使用 API Key 认证，请将密钥放在请求头 x-api-key 中。
2. 不要改用 OAuth，也不要写成 Authorization: Bearer <API Key>。
3. 请根据当前客户端支持的 MCP 配置方式自行安装；完成后按需刷新工具列表或重启客户端。
4. 请验证服务已连接并能读取工具列表，只向我报告配置位置、测试结果和后续用法，不要在回复中回显完整 API Key。
```

<Tip>
  Agent 若要求确认写入配置文件、重启客户端或执行连接测试，可以在核对目标路径和命令后授权。若它尝试启动 OAuth，请提醒它严格使用 `x-api-key` 请求头。
</Tip>

仅在可信的本地 Agent 或私密对话中粘贴真实 Key。Agent 不具备配置权限或自动安装失败时，改用下面的手动配置。

## 方式二：手动配置

### 通用 HTTP MCP 配置

```json
{
  "type": "http",
  "url": "https://mcp.bazhuayu.com",
  "headers": {
    "x-api-key": "你的APIKey"
  }
}
```

### Cursor 示例（`~/.cursor/mcp.json`）

```json
{
  "mcpServers": {
    "octoparse": {
      "type": "http",
      "url": "https://mcp.bazhuayu.com",
      "headers": {
        "x-api-key": "你的APIKey"
      }
    }
  }
}
```

### Claude Code 示例（`~/.claude/settings.json`）

```json
{
  "mcpServers": {
    "octoparse": {
      "type": "http",
      "url": "https://mcp.bazhuayu.com",
      "headers": {
        "x-api-key": "你的APIKey"
      }
    }
  }
}
```

保存后重启客户端，在 MCP 工具列表中应能看到八爪鱼相关工具。

更多客户端（ChatGPT、Gemini、OpenClaw、VS Code）见 [客户端配置指南](/zh/mcp/guides/clients)。

## 验证是否生效

连接成功后，可在对话中尝试：

- 「用八爪鱼搜索一个采集新闻标题的模板」
- 「列出我账户下的八爪鱼任务」

若返回 `401` / `403`，请检查 Key 是否有效、是否填在 `x-api-key` 字段，以及 URL 是否正确。
