<Note>
  上表用于说明覆盖的广度与代表性来源，不是固定白名单，也不表示每个平台的全部历史内容都可查询。平台接入、可检索时间范围和字段可能调整，应以 `list_platforms` 的实时返回及实际搜索结果为准。

## 参数

无需参数：

```json
{}
```

## 响应

返回 `success`、`code`、`message`、`data` 和 `timestamp`。平台条目至少包含用于识别来源的 `platform` 与 `domain`，服务还可能返回 `platformCode`、`platformName`、`sortOrder` 或总数等扩展字段。

<Note>
  当前服务声明的 Schema 与部分实时响应包装形式存在差异。集成时应以 `data` 中实际返回的平台条目为准，不要依赖固定的额外包装层或扩展字段。
</Note>

平台清单可能随数据中心能力变化。需要限制内容搜索范围时，把返回的域名传给 `search_platform_content` 的 `include` 或 `exclude`。
