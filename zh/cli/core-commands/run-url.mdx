> `octopus run-url` 是 v0.1.22 的快速采集入口。输入 URL 后，CLI 会自动识别页面结构并直接运行采集。

## `octopus run-url`（无任务直接采集）

**命令描述**

当你还没有八爪鱼任务模板，只想快速采集一个页面时，使用 `run-url --auto`。它适合探索性采集、新页面试采、AI Agent 调用和一次性数据提取。

## 基本用法

```bash
octopus run-url https://detail.tmall.com/item.htm?id=710033201461 --auto
```

带目标说明和条数限制：

```bash
octopus run-url https://movie.douban.com/explore --auto --goal "提取电影名称、评分、简介和豆瓣链接" --max-rows 20
```

选择特定候选方案：

```bash
octopus run-url https://example.com/products --select candidate_001
```

传入输入参数：

```bash
octopus run-url https://www.tmall.com --auto --goal "搜索并采集商品列表" --input keyword=烤面包机
```

## 结构化输出

```bash
# JSON 单次输出
octopus run-url URL --auto --json

# JSONL 流式输出，适合长时间任务或管道处理
octopus run-url URL --auto --jsonl
```

脚本、Codex 或 Agent 集成时，优先使用 `--json` / `--jsonl`，不要解析人类可读输出。

## `run-url` vs `run`

| 场景 | 推荐命令 |
| --- | --- |
| 新 URL，没配过任务模板 | `octopus run-url URL --auto` |
| 快速试采一个页面 | `octopus run-url URL --auto --max-rows 20` |
| 已有稳定任务，需要重复采集 | `octopus run <taskId>` |
| 已生成本地任务文件，需要精确控制 | `octopus run <taskId> --task-file task.json` |

## 与 `recognize` 的关系

- `run-url`：识别并直接采集，适合一步到位。
- `recognize`：先生成任务文件，适合校验、复用、人工确认或 Agent 计划编排。

## 常见注意事项

| 问题 | 建议 |
| --- | --- |
| 采集目标不明确 | 增加 `--goal "..."` |
| 数据量未知 | 调试时先加 `--max-rows 20` |
| URL 报错 | 确认包含 `https://` 或 `http://` |
| PowerShell 中文 goal 被拆分 | 用双引号包裹 `--goal` 的值 |
