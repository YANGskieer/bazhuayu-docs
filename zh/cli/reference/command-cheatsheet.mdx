> 八爪鱼 CLI 常用命令一览。尖括号 `<taskId>`、`<apiKey>`、`<url>` 表示需替换为实际值。

除特别说明外，功能命令均需先完成 `octopus auth login`。脚本、CI、Codex 或 Agent 调用时优先使用 `--json` / `--jsonl`。

## 帮助与诊断

```bash
octopus --help
octopus --version
octopus capabilities
octopus doctor
octopus browser status --browser-id chrome --json

octopus capabilities --json
octopus doctor --json
octopus doctor --output ./runs --api-base-url https://api.example.com --json
```

## 浏览器管理

```bash
octopus browser status --browser-id chrome --json
octopus browser profiles --browser-id chrome --json
octopus browser install --browser-id chrome --profile "Default" --force-close --json
octopus browser use user --browser-id chrome --profile "Default" --json
octopus browser use status --json
octopus browser use independent --json
octopus browser close --browser-id chrome --profile "Default" --json
```

用户浏览器模式仅支持 Windows 和 macOS，且不支持 `--headless`。安装扩展后需重新打开浏览器并确认 `browser status --json` 返回 `readyForUserBrowserRun: true`。

## 认证

```bash
octopus auth login
octopus auth login --oauth
octopus auth login <apiKey>
octopus auth login --stdin
octopus auth login --no-open
octopus auth status
octopus auth status --json
octopus auth logout
```

环境变量：

```bash
OCTOPUS_API_KEY=<apiKey> octopus task list --json
OCTOPUS_ACCESS_TOKEN=<accessToken> octopus task list --json
OCTOPUS_API_BASE_URL=https://example.com octopus task list --json
octopus auth status --api-base-url https://example.com --json
```

## 网页检测与任务生成

```bash
octopus detect <url>
octopus detect <url> --agent --agent-command "node make-plan.mjs" --goal "提取商品信息" --run-sample 5 --json
octopus detect <url> --agent --agent-command "node make-plan.mjs" --goal "提取商品信息" --run-sample 5 --run-output ./sample-runs --keep-agent-files --json
```

Agent 审计流程：

```bash
octopus detect <url> --prepare-agent --json --goal "提取用户评论" --output context.json
octopus detect --preview-agent-plan plan.json --agent-context context.json --json
octopus detect --apply-agent-plan plan.json --agent-context context.json --output task.json --json
```

用户直接操作 CLI：

```bash
octopus detect <url> --auto --output task.json
octopus detect <url> --manual --output task.json
octopus detect <url> --auto --goal "提取商品评论" --output task.json
octopus detect <url> --auto --goal "提取商品信息" --llm-rank --output task.json
octopus detect <url> --auto --no-dismiss-popups --output task.json
octopus detect <url> --select protected_smart_1 --output task.json
octopus detect <url> --auto --query "烤面包机" --output task.json
octopus detect <url> --manual --save-session --session-name shop --output task.json
octopus detect <url> --auto --browser user --browser-id chrome --profile "Default" --output task.json
```

`--agent-command` 必须是可信的本地 shell 命令；自然语言目标使用 `--goal`。`--run-sample` 只支持 `--agent`，且参数必须是正整数。`detect --agent` 不需要 `--yes`；该参数仍为旧脚本兼容保留。

## 本地采集

```bash
octopus run <taskId>
octopus run <taskId> --headless
octopus run <taskId> --max-rows 100
octopus run <taskId> --detach
octopus run <taskId> --output ./runs
octopus run <taskId> --chrome-path /path/to/chrome
octopus run <taskId> --browser independent --headless
octopus run <taskId> --browser user --browser-id chrome --profile "Default"

octopus run <taskId> --task-file ./task.json --output ./results/
octopus run <taskId> --task-file ./task.json --detach --max-rows 100
```

浏览器选择优先级：`--browser` > `OCTOPUS_BROWSER` > `octopus browser use` > `independent`。用户模式下不要传 `--headless`；`--force-close-browser` 仅用于浏览器或 Profile 冲突时的故障恢复。

## 云采集与云定时

```bash
octopus cloud start <taskId>
octopus cloud stop <taskId>
octopus cloud status <taskId>
octopus cloud history <taskId>

octopus schedule cloud get <taskId> --json
octopus schedule cloud update <taskId> --type daily --date 1 --time 09:00 --yes --json
octopus schedule cloud start <taskId> --yes --json
octopus schedule cloud stop <taskId> --yes --json
octopus schedule cloud next --type daily --date 1 --time 09:00 --json
```

云定时类型：`date` / `once`、`weekly`、`monthly`、`interval-minute`、`every-hour`、`daily`，也可传 `1` 到 `6`。

## 数据历史、预览与导出

```bash
octopus data history <taskId>
octopus data history <taskId> --local
octopus data history <taskId> --cloud

octopus data count <taskId> --source local --json
octopus data count <taskId> --source cloud --json
octopus data count <taskId> --source cloud --unexported --json

octopus data preview <taskId> --source local --limit 20 --json
octopus data preview <taskId> --source cloud --limit 20 --offset 0 --json
octopus data preview <taskId> --source cloud --unexported --limit 20 --json

octopus data export <taskId> --format xlsx --file result.xlsx
octopus data export <taskId> --local --format csv --file result.csv
octopus data export <taskId> --lot-id <lotId> --format json
octopus data export <taskId> --source cloud --unexported --format csv --file unexported.csv
```

导出格式：`xlsx`、`csv`、`html`、`json`、`xml`。`--unexported` 读取云端未导出数据，但不会标记为已导出。
