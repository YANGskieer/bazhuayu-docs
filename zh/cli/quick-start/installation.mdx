> 通过 npm 安装八爪鱼 CLI，并确认 `octopus` 命令、浏览器依赖与本地运行环境可用。

八爪鱼 CLI 以 npm 包形式发布。v0.1.30 新增用户浏览器模式，并将本地运行时升级为 `@octopus/browser-runtime`；既有 Agent、任务管理、云定时和数据检查接口保持兼容。

## 环境要求

安装前请确认：

- Node.js **20 或更高**（推荐 22 LTS）
- npm **8 或更高**
- 可用终端：PowerShell、CMD、macOS 终端或 Linux Shell
- 默认独立模式由 CLI 管理 Chrome for Testing；用户模式需要系统已安装 Chrome 或 Edge

检查版本：

```bash
node -v
npm -v
```

## 平台支持

| 平台 | 独立浏览器本地采集 | 用户浏览器模式 | 云端采集 | 备注 |
| --- | :---: | :---: | :---: | --- |
| Windows x64 | 支持 | 支持 Chrome / Edge | 支持 | Windows 10 / 11 |
| macOS x64 | 支持 | 支持 Chrome / Edge | 支持 | 需 Node.js 20+ |
| macOS arm64 | 支持 | 支持 Chrome / Edge | 支持 | 需 Node.js 20+ |
| Linux x64 | 支持 | 不支持 | 支持 | 用户模式依赖桌面系统扩展安装能力 |
| Linux arm64 | 暂不支持 | 不支持 | 支持 | Chrome for Testing 暂无 Linux arm64 浏览器包 |

## 安装 Node.js

### Windows

1. 访问 [Node.js 官网](https://nodejs.org/) 下载 LTS 安装包。
2. 使用默认选项安装，并确认勾选加入 `PATH`。
3. 打开 PowerShell 或 CMD，执行：

```bash
node --version
npm --version
```

### macOS

可从 [Node.js 官网](https://nodejs.org/) 下载 macOS 安装包，或使用 Homebrew：

```bash
brew install node
node --version
npm --version
```

### Linux / 服务器

推荐使用 [nvm](https://github.com/nvm-sh/nvm) 安装 Node.js 20+：

```bash
node -v
npm -v
```

Linux x64 从 CLI v0.1.22 起支持本地采集；Linux arm64 目前仅建议使用云采集相关能力。

## 安装八爪鱼 CLI

安装或更新到 v0.1.30：

```bash
npm install -g bazhuayu-cli@0.1.30
```

安装完成后，命令名为：

```bash
octopus
```

查看版本与帮助：

```bash
octopus --version
octopus --help
```

v0.1.30 应能看到 `browser`、`detect`、`task-group`、`template`、`schedule` 和 `data preview` 等命令：

```bash
octopus browser --help
octopus detect --help
octopus task-group --help
octopus data preview --help
```

## 运行环境诊断

```bash
octopus doctor
octopus browser status --browser-id chrome --json
```

`doctor` 会检查 Node.js、本地运行时、Chrome for Testing 和认证/API 环境，并在 Windows / macOS 上报告可选用户浏览器的扩展状态。`browser status --json` 用于检查 Chrome / Edge、Profile 与扩展是否达到 `readyForUserBrowserRun: true`。

## 更新与卸载

更新到指定版本：

```bash
npm install -g bazhuayu-cli@0.1.30
octopus --version
```

卸载：

```bash
npm uninstall -g bazhuayu-cli
```

## 常见问题

| 现象 | 处理建议 |
| --- | --- |
| `octopus: command not found` | 将 Node.js 目录和 npm 全局 bin 目录加入系统 `PATH` |
| PowerShell 中文参数被拆开 | 给中文参数加双引号，例如 `--goal "提取电影名称、评分"` |
| PowerShell 多行命令报错 | 使用反引号 `` ` `` 续行，或优先写成单行 |
| URL 无法打开 | 检查是否写完整协议，例如 `https://example.com` |
| `--output task.json` 写到意外目录 | 先 `cd` 到工作目录，或使用绝对路径 |
| `npm install -g ...@latest` 卡住或被终止 | 使用固定版本：`npm install -g bazhuayu-cli@0.1.30` |
| `doctor` 某项失败 | 根据 `ok: false` 的检查项修复 Node.js、本地运行时或浏览器依赖 |
| 用户浏览器扩展未就绪 | 按 [`browser` 首次配置流程](/zh/cli/core-commands/browser-management) 安装扩展、重开浏览器并再次检查状态 |
