# 云排错（团队版及以上参考）

本教程针对本地采集有数据、云采集没有数据的排错教程。如果本地采集没有数据，请先参考[本地排错教程](/zh/academy/troubleshooting/local-debug)。

本地采集有数据、云采集没有数据的主要原因：

1. 防采集
2. 网站或网速原因
3. 网络环境不同，源码有变，原 XPath 定位不准
4. 网站只允许单浏览器或单 IP 登录

## 一、防采集

防采集主要是以下三类：IP 被封禁止访问、出现验证码、云上需要登录。

针对以上三种情况，都可以通过采集网页的 HTML 源码进行观察。例如启动云采集后子任务出现采集为 0 的情况，可以采集网页源码观察网站在云上打开的情况。

### 采集网页源码的步骤

1. 在规则对应位置增加【提取数据】步骤，然后添加字段采集当前网页的源码。
2. 启动云采集查看云采集数据，将采集到的源码复制粘贴到 txt 文件，然后另存为 `.html` 文件。
3. 将另存的 `.html` 文件在浏览器中打开查看。

<Note>
  - 如果是 IP 被封禁止访问、云上需要登录的情况，也可以通过以上方法采集网页源码后在浏览器里查看。IP 被封可以尝试设置切换 IP 尝试采集。
  - 如果是需要登录则在规则里设置登录。
  - 规则中使用了 Cookie 登录，本地采集的时候可正常调用本地缓存的 Cookie。启动云采集后，云采集使用云端服务器节点，是新的 IP，没有登录 Cookie，不能以登录状态打开要采集的网页，也需要重新设置登录的步骤。
  - 有的网站不支持多 IP 登录，此类网站的采集规则，需勾选【云采集不拆分任务】。
</Note>

## 二、网站或网速原因

如果网站未完全打开，一个完全打开的网站应该看到完整的页面内容，而非加载中的图标。

有时候一个网页即便完全打开了，列表数据也要延迟几秒才能加载出来。网页已经处于完全打开状态，但列表数据还没有加载出来。在八爪鱼中，如果一个操作已经完全执行完成（例如网页完全打开），它会立即执行下一个操作步骤，但此时列表数据没有加载出来，所以执行失败，云采集没有数据。

**解决措施：** 碰到这种因网速原因或网站本身原因导致数据加载过慢时，需要进行规则优化——设置合适的执行前等待时间。

## 三、网络环境不同，源码有变，原 XPath 定位不准

部分网站在不同环境下打开网页源码会有变化，从而导致本地采集正常但云上 XPath 定位不准导致采集不到数据。

**解决方法：**

1. 用前面采集网页 HTML 源码的方法，采集网页的源码后保存为 `.html` 文件，然后在浏览器中打开文件。
2. 根据云上打开的网页源码修改字段的 XPath。

> 修改 XPath 需要学习掌握 [XPath 的知识](https://www.bazhuayu.com/helpcenter/docs/NNq78SWX)。

## 四、网站只允许单浏览器或单 IP 登录

网站只允许单浏览器或单 IP 登录，如果任务进行了拆分会导致云采集不到数据。

**解决方法：** 设置云采集不拆分。

## 五、规则问题——增量采集

**原因：** 云采集具有增量采集功能：根据 URL 判断是否采集过，第一次采集后，后续不会再采集已采集过的 URL。如果是列表信息采集，如果后续有增加新增列表但 URL 没有发生改变，八爪鱼会跳过此条 URL 不采集。

**解决方法：** 关闭增量采集。

## 六、快速排错的小技巧

### 1、提取数据增加提取网页页码

帮助我们快速定位数据采集从哪一页开始出现问题。

### 2、提取数据增加提取网页网址

提取不到字段时，帮助我们快速检查网页情况——是网页本身的问题（网址失效、本身无此字段），还是八爪鱼问题（无法正常打开网页、XPath 定位不准等）。

### 3、提取数据增加提取网页源码

常见于云采集，便于我们查看采集任务在云上的运行环境。
