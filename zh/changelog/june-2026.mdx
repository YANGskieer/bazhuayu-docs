# 2026 年 6 月发布说明

> **更新日期：** 2026 年 6 月 10 日  
> CLI 国内版本更新至 v0.1.22；MCP 五大 AI 客户端完整中文对接教程上线；站点根路径与更新日志入口路由优化。

## 新功能

### CLI 国内更新 v0.1.22

本次 CLI 从 `0.1.14` 升级到 `0.1.22`。安装或更新：

```bash
npm install -g bazhuayu-cli@0.1.22
```

新增能力：

- [`octopus recognize`](/zh/cli/core-commands/recognize) — 从网页 URL 识别结构并生成任务文件
- [`octopus run-url`](/zh/cli/core-commands/run-url) — 无需预制任务，直接从 URL 自动识别并采集
- 支持列表页、详情页、列表 + 详情页任务与分页识别
- 新增 Agent 模式上下文准备与计划应用流程
- 新增 Linux x64 平台本地采集支持

### MCP 服务 — AI 客户端完整对接教程

ChatGPT、Claude、Cursor、VS Code、Gemini 五个平台的 MCP 对接页由外部跳转改为 **站内完整中文分步指南**，每篇均含：

- 前置条件、MCP 服务地址（`https://mcp.bazhuayu.com`）与 OAuth 认证说明
- 分步操作截图（安装插件 / 添加 MCP 服务器 / 登录授权等）
- 配置示例（如 Cursor / VS Code 的 `settings.json`、Gemini CLI 的 OAuth `clientId` 等）
- 接入后可用的能力说明与常见问题

| 平台 | 文档 |
| --- | --- |
| ChatGPT | [MCP 对接 ChatGPT](/zh/mcp/integrations/chatgpt) |
| Claude | [MCP 对接 Claude](/zh/mcp/integrations/claude) |
| Cursor | [MCP 对接 Cursor](/zh/mcp/integrations/cursor) |
| VS Code | [MCP 对接 VS Code](/zh/mcp/integrations/vscode) |
| Gemini | [MCP 对接 Gemini](/zh/mcp/integrations/gemini) |

各教程统一说明：导出格式为 **EXCEL / CSV / JSON / 数据库**；云采集可能消耗八爪鱼账户余额，具体取决于所用模板。

## 改进

### CLI 命令与文档

- `octopus run` 支持 `--task-file`、`--detach`、`--max-rows`、`--headless`
- `octopus task validate` 支持校验模板任务与本地任务文件
- `octopus data history/export` 作为本地与云端数据访问的统一入口
- 认证支持 API Key、OAuth、stdin、环境变量与自定义 API 地址
- 同步更新 [CLI 概述](/zh/cli/index)、[安装](/zh/cli/quick-start/installation)、[登录](/zh/cli/quick-start/get-api-key-and-log-in)、[运行第一个任务](/zh/cli/quick-start/run-your-first-task)、[命令速查表](/zh/cli/reference/command-cheatsheet) 等页面

### 站点路由

- 根路径 **`/`** 自动跳转至 [概述](/zh/overview)
- **`/zh/changelog`** 自动跳转至最新月份更新日志

### 文档内容规范

- MCP 对接教程统一使用八爪鱼品牌表述，移除 Octoparse 相关外链与重复说明
- 各 AI 客户端页采用与 Coze / QClaw 一致的分步教程版式

## 注意事项

- `recognize` 与 `run-url` 仍需可用 Chrome 环境
- Linux arm64 暂不支持本地执行，可使用云采集能力
- PowerShell 中中文 `--goal` 参数请使用双引号包裹
